# Oh-My-Brain

An Obsidian-based "second brain" vault — personal knowledge management, web clippings, AI agent workspace orchestration, and plugin development.

**Author:** Xu Jihui
**License:** MIT
**Language:** Chinese for conversation, English for code & configuration

## Directory Structure

```
oh-my-brain/
├── Brain/                  # Core knowledge: SOUL.md, Memory.md, etc.
├── Clippings/              # Web clippings: articles & pages collected from browsers
├── Skills/                 # Workspace skills: skill files for the current workspace
├── Agents/                 # Agent workspace management (see details below)
│   └── <alias>/            # Each subdirectory = one agent workspace
│       └── CLAUDE.md       # → Symlink to external repo's CLAUDE.md
├── Output/                 # Agent output: generated reports, slides, images, etc.
├── oh-my-brain-plugin/     # Plugin source, setup script, and shared config
│   ├── src/                # TypeScript source code
│   ├── folder-colors.json  # Shared folder color config (git-tracked)
│   └── setup.py            # One-click init script
├── .obsidian/              # Obsidian internal config — do not modify casually
│   └── plugins/            # Compiled plugin output directory
├── .claude/                # Claude Code configuration
└── CLAUDE.md               # ← You are reading this file
```

### Brain/ — Core Knowledge

Stores the agent's identity and memory — the "soul" of the vault.

- `SOUL.md` — Agent identity, behavioral constraints, and coordination rules
- `Memory.md` — Persistent runtime memory: lessons learned, decisions, user preferences
- Files in this directory are critical — **confirm user intent before modifying**

### Clippings/ — Web Clippings

Articles, notes, and references collected from browsers or other sources.

- Files typically include YAML frontmatter (source URL, date, tags, etc.)
- Can be added via Obsidian Web Clipper or manually
- Preserve original content integrity; do not break source references when editing

### Skills/ — Workspace Skills

Skill definition files available to the current workspace.

- Skill files follow the standard format (name, description, content)
- Loaded on demand to provide domain-specific capabilities

### Output/ — Agent Output

Generated artifacts produced by the Oh-My-Brain agent during tasks.

- Markdown reports and summaries (`.md`)
- Slide decks in Marp format (`.md` with Marp frontmatter)
- Images generated by matplotlib or other tools (`.png`, `.svg`, etc.)
- This directory is **write-friendly** — the agent can freely create files here without user confirmation
- Organize by subdirectory when output volume grows (e.g., `Output/slides/`, `Output/charts/`)

### Agents/ — External Agent Workspaces

External code repositories managed via the Oh My Brain plugin. Each subdirectory represents an independent agent workspace.

- Subdirectory name = agent alias (e.g., `Agents/my-project/`)
- Each subdirectory contains a **symlink** `CLAUDE.md` pointing to the external repo's CLAUDE.md
- Symlinks are created automatically by the plugin (symlink preferred; falls back to copy)
- Plugin configuration is stored in `.obsidian/plugins/oh-my-brain/data.json`
- **Note:** CLAUDE.md files in agent subdirectories are links to external files; their content belongs to external projects

## Plugins

### Oh My Brain Plugin

In-house Obsidian plugin for agent management, folder coloring, and external repo integration.

| Item | Description |
|------|-------------|
| Source | `oh-my-brain-plugin/` (TypeScript) |
| Build output | `.obsidian/plugins/oh-my-brain/` |
| Plugin ID | `oh-my-brain` |
| Build command | `cd oh-my-brain-plugin && npm install && npm run build` |

**Core features:**
1. **Folder coloring** — Right-click any folder to set color (27 presets + custom); supports cascade to children
2. **Shared color config** — `oh-my-brain-plugin/folder-colors.json` (git-tracked); user overrides in `data.json` (gitignored)
3. **Add External Agent** — Add an agent via command palette: select local repo path + alias + color
4. **CLAUDE.md linking** — Auto symlink (falls back to copy); supports manual Sync

### Claudian Plugin

Third-party plugin providing Claude AI chat integration within Obsidian.

- GitHub: `YishenTu/claudian`
- Optionally downloaded via `setup.py`
- Requires Claude API Key configuration

## Setup

```bash
# One-click init: vault + build plugins + optional remote plugin download
python oh-my-brain-plugin/setup.py

# Common options
python oh-my-brain-plugin/setup.py --force          # Force rebuild / reinstall
python oh-my-brain-plugin/setup.py --skip-build     # Skip local plugin build
python oh-my-brain-plugin/setup.py --skip-download  # Offline mode, skip remote plugin download
```

After init: Obsidian → Settings → Community plugins → Disable Restricted mode → Enable plugins.

## Conventions

### File Format
- All notes use Markdown (.md)
- YAML frontmatter for metadata — respect existing fields, do not alter arbitrarily
- Internal links use Wiki-link format: `[[note-name]]` or `[[folder/note-name]]`

### Path Rules
- Use **relative paths** (from vault root) for vault files
- Absolute paths only for external contexts (e.g., an agent's external repo path)
- Export paths: `~/Desktop`, `~/Downloads`

### Safety Principles
- **Never overwrite content you have not read** — Read first, then Edit
- **Confirm user intent before modifying files under `Brain/`**
- **`CLAUDE.md` under `Agents/` are symlinks** — modifying them affects external repos; do not modify unless explicitly requested
- **Do not touch `.obsidian/` config** unless you clearly know what you are doing
- **Do not break Dataview queries** (` ```dataview` ` ` code blocks)

### Development Standards
- Plugin development uses TypeScript + esbuild
- Windows environment (Git Bash / MINGW64); npm commands may require `shell=True`
- Build artifacts output to `.obsidian/plugins/<plugin-id>/`
